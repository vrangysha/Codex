# Постсервисный бот для клиник

MVP для стоматологий, косметологий и ветклиник: сопровождение после процедуры, контроль состояния, эскалация тревожных симптомов, отзывы и повторные визиты.

## Важное ограничение

Бот не лечит, не ставит диагноз и не заменяет врача. Он отправляет утвержденные клиникой памятки, собирает состояние и передает тревожные случаи администратору/врачу.

## MVP

- Шаблоны протоколов по услугам.
- Расписание сообщений 6/24/72 часа.
- Анкеты состояния.
- Загрузка фото.
- Красные флаги и эскалация.
- Напоминание о контрольном визите.
- Сбор отзыва.

## Старт реализации

В `src/postservice_clinic_care/workflow.py` реализовано доменное ядро: кейс наблюдения, расписание, чек-ин, красные флаги и эскалация.
В `src/postservice_clinic_care/protocols.py` реализован каталог утвержденных протоколов: памятки, вопросы, расписания, красные флаги и создание кейса по выбранному протоколу.
В `src/postservice_clinic_care/escalation.py` реализована очередь эскалаций: создание тревожного тикета, взятие в работу, закрытие и история действий.
В `src/postservice_clinic_care/scheduler.py` реализованы due-чек-ины и message adapter без медицинской генерации.

## Delivery

- `demo.py` - локальный сквозной сценарий.
- `RUNBOOK.md` - запуск, тесты, внедрение и safety rules.
- `INTEGRATION_CONTRACT.md` - контракт CRM/МИС/мессенджеров.
- `.env.example` - переменные для реального внедрения.
