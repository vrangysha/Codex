# 00 — Как пользоваться этим набором prompt-файлов

## Назначение

Этот набор нужен, чтобы Codex поэтапно сделал полноценный многостраничный лендинг для бизнеса ремонта квартир и домов: от фундамента проекта и дизайн-системы до отдельных страниц, форм, SEO, изображений, адаптива и финального QA.

## Как запускать

1. Открой проект в Codex.
2. Убедись, что проект чистый или ты понимаешь текущий diff.
3. Вставляй в Codex содержимое файлов по порядку:
   - `01_PROJECT_BOOTSTRAP_PROMPT.md`
   - `02_AGENTS_AND_GLOBAL_RULES_PROMPT.md`
   - `03_DESIGN_SYSTEM_PROMPT.md`
   - и так далее.
4. Не запускай следующий этап, пока предыдущий не завершён и Codex не обновил `WORKLOG.md`.
5. После каждого крупного этапа проси Codex запускать доступные проверки: `lint`, `typecheck`, `build`, `test`.

## Рекомендуемый режим работы

Работай маленькими итерациями:

- этап 1–5: фундамент, дизайн-система, данные, общие компоненты;
- этап 6–18: страницы сайта;
- этап 19–23: юридические страницы, изображения, SEO, формы;
- этап 24–26: адаптив, QA, финальная сдача.

## Если проект пустой

Codex должен создать Next.js + TypeScript + Tailwind CSS проект или использовать существующий шаблон, если он уже есть.

## Если проект уже существует

Codex не должен ломать текущий стек. Он обязан изучить структуру проекта, использовать существующие компоненты и стили, а новые изменения делать аккуратно.

## Подключение скиллов

Перед работой желательно установить:

```bash
npx impeccable skills install
```

После перезапуска Codex:

```text
/impeccable init
```

Для UI UX Pro Max используй установку, указанную в его репозитории. Если skill недоступен в Codex, Codex должен вручную применить его принципы: структура, визуальная иерархия, UX, accessibility, дизайн-система, адаптив, проверка интерфейса.

## Использование 21st.dev

Codex может искать и использовать бесплатные публичные компоненты 21st.dev, особенно для:

- navigation;
- hero;
- feature cards;
- pricing;
- testimonials;
- FAQ;
- forms;
- footer;
- marketing sections.

Правило: любые установленные компоненты нужно привести к единому стилю сайта и проверить.

## Главный критерий

Итог должен быть не набором случайных страниц, а цельным коммерческим сайтом с понятной архитектурой, светлой эстетикой, хорошим UX и проверенным build.
