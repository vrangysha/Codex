# Codex Prompt Pack: многостраничный лендинг ремонта квартир и домов

Этот файл — отдельный этап работы для Codex. Запускай файлы строго по порядку. Перед началом каждого этапа попроси Codex прочитать уже созданные `AGENTS.md`, `DESIGN.md`, `CONTENT_TODO.md`, `ASSETS.md`, `QA_REPORT.md`, `WORKLOG.md` и текущую структуру проекта.

Общие правила для всех этапов:
- Не делай git commit без отдельного разрешения.
- Не используй платные или закрытые компоненты 21st.dev.
- Не копируй проприетарный код и изображения без лицензии.
- Если данных бизнеса нет, используй аккуратные placeholder-данные и заноси их в `CONTENT_TODO.md`.
- Не выдавай сгенерированные изображения, отзывы и кейсы за реальные.
- После каждого этапа обновляй `WORKLOG.md` и `QA_REPORT.md`.
- Если build/lint/typecheck падают — исправляй, затем запускай повторно.
- Сайт должен быть современным, лаконичным, светлым, доверительным, адаптивным и SEO-готовым.

# 13 — Страница `/portfolio`

## Цель страницы

Показать работы, стиль, масштаб, процесс и результат. Если реальных работ нет, создать аккуратную структуру для будущего портфолио без обмана пользователя.

## Структура

### 1. Hero

- H1: “Портфолио ремонтов”;
- подзаголовок: примеры объектов, этапов и решений;
- CTA: “Рассчитать похожий проект”;
- secondary CTA: “Обсудить задачу”.

Если кейсы демонстрационные, добавь внутреннюю пометку в data, но в UI лучше использовать нейтральное “Примеры оформления кейсов” только если сайт ещё в демо-режиме.

### 2. Filters

Фильтры:

- Все;
- Квартиры;
- Дома;
- Косметический;
- Капитальный;
- Дизайн-проект;
- Ванные;
- Кухни.

Фильтры должны работать без ломания accessibility. Если реализация сложная, можно сделать server/static filter links.

### 3. Project grid

Карточка проекта:

- изображение;
- title;
- type;
- area;
- duration;
- budget range placeholder;
- short description;
- tags;
- CTA “Смотреть детали” или раскрытие.

### 4. Featured case

Один крупный кейс:

- задача;
- что сделали;
- результат;
- сроки;
- площадь;
- 3–4 фото/placeholder;
- CTA.

### 5. Before/After

Если нет реальных фото, создай component placeholder:

- “До”;
- “После”;
- skeleton image;
- пометка в `CONTENT_TODO.md`.

Не придумывай фальшивые before/after.

### 6. Scope examples

Покажи, что в проектах может быть:

- демонтаж;
- электрика;
- сантехника;
- стены;
- пол;
- плитка;
- мебельные закладные;
- освещение.

### 7. CTA

“Хотите похожий ремонт? Пришлите площадь и фото — подготовим предварительный расчёт.”

Форма короткая:

- имя;
- телефон;
- тип объекта;
- площадь;
- комментарий.

## SEO

- title: “Портфолио ремонтов квартир и домов”;
- description: “Примеры ремонтов квартир и домов: площади, сроки, состав работ и решения для разных объектов.”
- Schema: CollectionPage + BreadcrumbList.

## Asset rules

Каждое изображение:

- alt;
- source in `ASSETS.md`;
- optimized;
- не нелицензированное.

## Проверки

- фильтры работают;
- изображения имеют alt;
- нет claims “реальные работы”, если demo;
- mobile grid удобен;
- CTA работает.

## Критерии готовности

- портфолио выглядит доверительно;
- демо-данные не маскируются под реальные;
- структура готова для замены на реальные кейсы.
