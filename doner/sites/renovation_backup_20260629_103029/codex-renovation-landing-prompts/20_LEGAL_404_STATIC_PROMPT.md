# 20 — Юридические страницы, 404 и статические состояния

## Цель этапа

Создать обязательные служебные страницы и не оставлять неработающие маршруты.

## 1. `/privacy`

Создай страницу политики конфиденциальности placeholder.

Содержание:

- кто обрабатывает данные placeholder;
- какие данные собираются через формы;
- цель обработки;
- хранение;
- передача третьим лицам placeholder;
- cookies/analytics placeholder;
- права пользователя;
- контакты;
- дата обновления placeholder.

Важно:

- не выдавать текст за юридически проверенный;
- добавить вверху или внизу примечание: “Текст является шаблоном и требует проверки юристом”.
- занести в `CONTENT_TODO.md`.

## 2. `/terms`

Создай страницу условий использования placeholder.

Содержание:

- общие положения;
- информация на сайте;
- заявка не является договором;
- цены ориентировочные;
- интеллектуальная собственность;
- ответственность;
- контакты;
- дата обновления placeholder.

Важно:

- цены на сайте не публичная оферта, если владелец не подтвердил обратное;
- текст требует юридической проверки.

## 3. `not-found.tsx`

Создай красивую 404 страницу:

- заголовок “Страница не найдена”;
- текст;
- кнопка на главную;
- кнопка на услуги;
- CTA на консультацию;
- светлый стиль.

## 4. Loading / error states

Если стек поддерживает:

- создай `loading.tsx` для routes, где уместно;
- создай аккуратные error states для формы;
- не делай драматичные ошибки.

## 5. Metadata

Для `/privacy` и `/terms`:

- title;
- noindex, если уместно;
- canonical, если используется.

## Проверки

- ссылки на privacy/terms из footer работают;
- checkbox согласия в формах ведёт на `/privacy`;
- 404 открывается;
- нет юридически опасных обещаний.

## Критерии готовности

- служебные страницы есть;
- footer корректно ссылается;
- юридические placeholder отмечены.
